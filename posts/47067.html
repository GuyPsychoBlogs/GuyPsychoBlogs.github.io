结合OD查看32位汇编代码

入门到秃头

个人笔记

寄存器

cpu上的一个元（部件），读写速度比内存还要快

数据寄存器

一般用来 保存操作数，计算结果

EAX(Accumulator):累加寄存器，也称之为累加器
EBX (Base):基地址寄存器
ECX(Count):计数寄存器
EDX(Date):数据寄存器

32位寄存器：EAX,EBX,ECX,EDX
16位寄存器: AX,BX,CX,DX
8位寄存器：AL/AH BL/BH/CL/CH/DL/DH 也叫低位/高位寄存器 Low/High

指针寄存器

操作栈的寄存器
栈：保存 函数的参数，局部变量等
EBP:栈底的指针
ESP:栈顶的指针

32位寄存器出入栈

Push EAX //将EAX的值放到栈中 ESP 寄存器 -4

原先栈顶的数据会往下一格、

EAX的数据会放到栈顶

Pop EAX //将EAX的值从栈中取出 ,ESP寄存器 +4

栈顶的数据被取出放入 EAX

16位寄存器出入栈

Push AX //将AX的值放到栈中 SP 会 -2

原先栈顶的数据会往下一格、

AX的数据会压到栈顶

Pop AX //将AX的值从栈中取出 ,SP寄存器 +2

栈顶的数据被取出放入 AX

SS:SP //SS是段地址 SP是偏移地址
对于栈顶的段地址，其是存放在段寄存器 SS 中的，而对于栈顶的偏移地址，其则是存放在 SP 寄存器中的 。

记住，在任何时刻，SS:SP 都是指向栈顶元素 。

变址寄存器

ESi和EDi：用来存放一个地址的寄存器

指令指针寄存器

Eip ，指向CPU下次将要执行的代码地址 注意是将要执行的代码地址而不是以及执行过了

标志寄存器

flag寄存器16位

在od中的位置

EFLAGS 寄存器中的ZF寄存器

zf中的z是zero的意思，即zero flag 其作用就是判断运算结果与零是否相等，相等及位真（1），反之位假（0）

简单来说所有对操作数 进行运算和逻辑运算的指令，都会根据运算结果修改ZF标志

第一点：
大多数情况下，运算结果对ZF的修改，由结果是否为0决定

若运算结果为0，则ZF为1

若运算结果非0，则ZF为0

举一些例子

加法指令：比如ADD,ADC,INC等
减法指令：比如sub ssb CMP DEC NEG等
逻辑运算指令：比如SHL,SHR，SAL，RCL，RCR等，（ROL，ROR不会改变ZF，原因自己分析）
调整指令：比如AAA,AAS等
串指令中含有算数运算的那些。比如CMPS,SCAS等
控制转移指令中含有 算术运算的 比如LOOP

在dosbox 中显示的结果

OF <—–> OV(1) ; NV(0)
DF <—–> DN(1) ; UP(0)
IF <—–> EI(1) ; DI(0)
SF <—–> NG(1) ; PL(0)
ZF <—–> ZR(1) ; NZ(0)
AF <—–> AC(1) ; NA(0)
PF <—–> PE(1) ; PO(0)
CF <—–> CY(1) ; NC(0)

参考连接

段寄存器:是为了对内存分段管理而设置的，计算机需要对内存分段，以分配给不同的程序使用。

例：
指令：Mov dword ptr ds:[0x405528]，ebx

实际上相当于 将 ebx 的值移动到了 ds.base +0x405528 组合的地址中

段寄存器的结构图

不可见部分这段中的 32位Base 相当于 ds.base 例如 段寄存器分配 一段内存地址 是1——10 那么她就代表1起始地址

不可见部分这段中的 32位limt 表示 段寄存器 分配的一段内存的大小 例如 1——10 那么这一段内存大小就是10

不可见部分这段中的 16位Attribute 定义了 段寄存器分配的 这段内存地址 是否可读、可写、可执行

可见部分这段中的 16位 selecter 是 在下面OD中看到的寄存器的值， 也是全局符号表：有关windos 内核的内容。

在OD中只显示了 ES,CS,SS,DS,FS,GS这些段寄存器

汇编常用指令

数据传输指令： mov 指令交换寄存器数据，注意：交换的寄存器大小需要一样 16位与16位交换，32位与32位交换

例如：mov eax,ebx
不能 mov eax,bx 也不能 mov 123, EAX

加减运算指令： add （加） sub（减） 。 将相加或者相减 的结果存放到寄存器 注意：相加相减的寄存器要一样， 16位于16位相加减， 32位与32位相加减

例如
Add eax,ebx

Sub eax,ebx

不能 add eax,bx /sub eax,bx
也不能 add 123,eax/sub 123,eax

逻辑运算

逻辑运算指令：逻辑运算是先把值转换位二进制再按 位 比较运算 最后 比较运算 的结果转为 16进制

逻辑与

and指令 两个都为1就为1，只要有一个不为1 就 不为1

例：

把1H 和2H 换算成 二进制 经行比较运算

1H =01 2 H=10

              01
              10
            ———————
              00

二进制00 换算成十进制也是0

最终 mov eax的值 是 0

逻辑或

or指令 只要有一个为 1 就是1

例：

把1 和2 换算成 二进制 经行比较运算

1 H=01 2 H = 10

               01
               10
             ———————
               11

11换算成十进制为3

最终 EAX的值为3

逻辑异或

xor指令 相同为0 不相同为1

例

把十六进制 的 3 和 十六进制 6 换算成 二进制 经行比较运算

3H = 0011 6H = 0110

               0011
               0110
              ——————— 
               0101

0101 b = 5 （dec）

最终 eax 的值是5

逻辑非
not指令 如果是1则变成0 如果是0则变成1

例：

十六进制的 3 = 11
11 not = 1111111111111111111111111111111111111111111111111111111111111100

因为 二进制的11前面没有 数 可以看作0 所以 有这么多的0被not成了 1

在od 中 EAX 显示 FFFFFFFC //代表 11not

在 OD中的数字 16进制数 位于运算的时候 也是用 16进制的值来运算

移位指令

算数位移

算数左移

Mov eax,2 // 二进制的10b
假设它为 00000010 ==2

Sal eax,1 //算数左移与 逻辑左移 功能是一样的
假设它为 00000010==1

把 00000010 最高位0 移动到了CF标志位，最低位又加了个0得到

00000100 ==4

算数右移

Mov eax,2 //二进制的 10b
假设它为 00000010 ==2

Sar eax,1
假设它为 00000010==1

算数右移把 最低位的0给送入了 CF标志位 得到 0000001
不足 八位 在左边又添了个 0 0000001=> 00000001

逻辑位移

逻辑左移 SHL 与 算数左移一样

逻辑右移 SHR与算数右移 的区别 最高位补0

注意： 算数左移与逻辑左移一样， 但是 算是右移，和逻辑右移动不一样

Test 指令

test 指令：实际就是作逻辑与运算

test指令 与 逻辑与(and)不同点是：test指令不会改变值，只会改变Z标志位

And eax,1

Test eax,1

Cmp 指令

cmp 指令： 实际上就是减法运算 与算数运算（sub）不同点是：cmp指令不会改变值，只会影响到Z标志位

Sub eax,1

cmp eax,1

POP & push 指令

栈：保存 函数的参数，局部变量等
EBP:栈底的指针

ESP:栈顶的指针

栈的规则：先进后出，后进先出

Push ：压栈指令

Pop ：出栈指令

32位寄存器
Push EAX //将EAX的值放到栈中 ESP 寄存器 -4

原先栈顶的数据会往下一格、

EAX的数据会放到栈顶

Pop EBX //将EAX的值从栈中取出 ,ESP寄存器 +4
栈顶的数据被取出放入 EBX

16位寄存器
Push AX //将AX的值放到栈中 SP 会 -2

原先栈顶的数据会往下一格、

AX的数据会压到栈顶


Pop BX //将AX的值从栈中取出 ,SP寄存器 +2
栈顶的数据被取出放入 BX

SS:SP //SS是段地址 SP是偏移地址
对于栈顶的段地址，其是存放在段寄存器 SS 中的，而对于栈顶的偏移地址，其则是存放在 SP 寄存器中的 。

注意: 在任何时刻，SS:SP 都是指向栈顶地址 。

知道ss，sp求物理地址的公式是 物理地址=ss* 16+ sp，或者物理地址=ss* 10H+ sp 也就是对于2进制来说，ss里的数要加上4个0 再加sp得到物理地址。比如ss存放 1234H sp 存放 2000H ，那么物理地址就是 1234H*10H+2000H=14340H。 如果ss是2000H sp是1234H，那么物理地址就是 21234H

Jmp 指令

jmp指令:无条件跳转指令

nop指令

nop指令：空指令（CPU执行到此处什么也不干） 只会改变 EIP的值

JCC指令

Jcc指 条件跳转指令，cc就是指条件码

常见的JZ指令：如果表达式计算结果等于0，那么zf标志位会置为1，则，jz 指令跳转

jcc指令说明

call指令

call：
CALL指令调用一个子程序，CPU执行call指令，进行两步操作：
（1）将当前的 IP 或 CS和IP 压入栈中；
（2）转移到紧跟的标号行地址执行程序。
常用来调用函数；也可以调用一个过程(少见)

相当于下面步骤
1、jmp +函数地址
2、push call 函数指令的下一行指令的地址 并且esp -4

OD中 F7单步步入 进入call指令内部

retn指令

retn：
Retn 指令是返回到主程序 把栈内的数据 放到了eip 寄存器

相当于下面步骤
pop eip //把刚刚压入栈的值放到了 eip寄存器中

END
